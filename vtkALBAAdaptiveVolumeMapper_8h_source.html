42// .NAME vtkALBAAdaptiveVolumeMapper - direct rendering of volume data
43
44// .SECTION Description
45// vtkALBAAdaptiveVolumeMapper performs a fast volume rendering of volume data using adaptive (LOD) techniques
46// This class supports both 1D and 2D transfer functions (vtkALBATransferFunction2D). Also, unlike the standard VTK volume renderers, this class supports vtkRectilinearGrid objects.
47// It is currently assumed that X and Y spacing in volume data is constant. If this stops being the case, the code should be modified (see vtkALBADistanceFilter).
48//
49// Current limitations: the renderer supports multiresolution volumes to accelerate the rendering. This optimisation however is heavely dependent on the heuristics that are used to assign
50// level-of-detail to different blocks of the volume. The current heuristics are trivial and some work have to be done to improve them (see PrepareBlockLODTemplate). When this is done the rendering speed can
51// increase dramatically..
52
53// .SECTION see also
54// vtkVolumeMapper vtkALBATransferFunction2D
55
