12// 'Wrapped Meter' (under Operations->Create->Drive) is for single object wrapping whereas 'Multi-obj Wrapped Meter' is for sphere-capped cylindrical wrapping.
13// Data requirements (for sphere-capped cylindrical wrapping program to work properly)
14//
15// 1. sphere radius > cylinder radius
16// 2. start/origin point of action line should be outside the sphere and end/insertion point should preferrably be on the cylinder surface or outside.
17// When end points is outside the cylinder surface, the program performs wrapping over sphere object only.
18// 3. The line conecting the start point and the end point must intersect at least one of the two objects for wrapping to occur. Otherwise (when it does not intersect any object), it will simply connect the two points by a straight line.
19// 4. "Wrapped sphere" (input button name in the interface) must be a parametric surface object of sphere type.
20// 5. "Wrapped cylinder" must be a parametric surface object of cylinder type.
21// 6. Start and end points can be either landmark vme or parametric surface vme.
22// 7. 'pathNum' in interface indicates total number of paths considered to find the shortest one. Too lower value may have effect on accuracy and too high will have effect on speed.Default value is set to 36.
23//
24// This program is based on follwoing paper with necessary extensions/modifications:
25// A. Audenaert and E. Audenaert, "Global optimization method for combined spherical-cylindrical wrapping in musculoskeletal upper limb modelling", Computer Methods and Programs in Biomedicine, 2008.
