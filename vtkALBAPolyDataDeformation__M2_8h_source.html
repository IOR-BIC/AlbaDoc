12 vtkALBAPolyDataDeformation_M2 performs a skeleton based deformation
13 of the input poly data set (mesh). The deformation is driven by
14 two skeletons, here aka the original and the deformed curves.
15 The original curve(skeleton) corresponds to the input (original)
16 mesh and it actually defines its structure. The filter parametrizes
17 every vertex of the mesh using this skeleton. The deformed curve
18 corresponds to the desired output mesh. The correspondence between
19 both curves can be either specified (fully or partially) or it
20 is computed automatically. The movement of skeleton nodes then
21 defines how the mesh should be deformed - parametrized vertices
22 are converted into Cartesian coordinates. Note that the mesh may
23 have more than one skeleton (this may be useful for complex
24 models that cannot be described by only one 1D skeleton in
25 the requested accuracy.
26
27 The deformation is based on:
28
29 Blanco FR, Oliveira MM: Instant mesh deformation.
30 In: Proceedings of the 2008 symposium on Interactive 3D graphics and games,
31 Redwood City, California, 2008, pp. 71-78
32
33 The correspondence detection between skeletons is primarily based on:
34
35 Rahmati M, Mirzaei A: Shape recognition by clustering and matching of skeletons.
36 Journal of Computers 2008, Academy Publisher, May 2008, 3(5):24-33
37
38 and
39
40 The Hungarian algorithm (http://en.wikipedia.org/wiki/Hungarian_algorithm)
41*/

652inline bool vtkALBAPolyDataDeformation_M2::
653CMunkres::FindUncoveredInMatrix(double item, int &row, int &col)
654{
655 for ( row = 0 ; row < matrix->GetNumberOfRows() ; row++ )
656 {
657 if ( !row_mask[row] )
658 {
659 for ( col = 0 ; col < matrix->GetNumberOfColumns() ; col++ )
660 {
661 if ( !col_mask[col] && (*matrix)(row,col) == item )
662 return true;
663 }
664 }
665 }
666
667 return false;
668}
